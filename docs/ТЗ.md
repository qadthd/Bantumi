## 1. Назначение и цели проекта

Целью проекта является разработка программной реализации настольной игры «Бантуми» (Kalah / Манкала) для запуска в веб-браузере без установки дополнительного программного обеспечения. Игра рассчитана на двух игроков, играющих на одном устройстве поочерёдно.

---

## 2. Требования к функциональности

### 2.1 Игровая логика
| № | Требование |
|---|-----------|
| Ф-01 | Инициализация доски с N камнями в каждой из 12 лунок (N = 3–6, задаётся игроком) |
| Ф-02 | Раскладывание камней против часовой стрелки с пропуском Калаха противника |
| Ф-03 | Бонусный ход при попадании последнего камня в свой Калах |
| Ф-04 | Захват камней при попадании последнего камня в пустую лунку своей стороны |
| Ф-05 | Определение конца игры и автоматический сбор оставшихся камней |
| Ф-06 | Подсчёт итогового счёта и определение победителя |
| Ф-07 | Отмена последнего хода (Undo) |

### 2.2 Пользовательский интерфейс
| № | Требование |
|---|-----------|
| Ф-08 | Визуальная доска с 12 лунками и 2 Калахами |
| Ф-09 | Отображение количества камней в каждой лунке в реальном времени |
| Ф-10 | Визуальная подсветка последнего хода и захвата |
| Ф-11 | Индикатор текущего хода (Игрок 1 / Игрок 2) |
| Ф-12 | Журнал ходов с отметкой бонусных ходов и захватов |
| Ф-13 | Оверлей с результатом по окончании игры |
| Ф-14 | Кнопки «Новая игра» и «Отменить ход» |
| Ф-15 | Настройка количества камней в лунке перед партией |

## 3. Нефункциональные требования

| Категория | Требование |
|-----------|-----------|
| Совместимость | Работает в Chrome, Firefox, Safari, Edge без установки |
| Адаптивность | Корректное отображение на экранах от 360px до 1920px |
| Надёжность | Состояние игры не теряется при нажатии кнопок интерфейса |
| Читаемость кода | Код разделён на независимые модули с комментариями |
| Доступность | Интуитивно понятный интерфейс без инструкций |
