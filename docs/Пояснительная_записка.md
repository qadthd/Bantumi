## 1. Описание предметной области

«Бантуми» (Kalah) — стратегическая настольная игра из семейства Манкала, широко распространённая в Африке. Игра рассчитана на двух участников и ведётся на доске с 12 малыми лунками и двумя хранилищами (Калахами).

### Правила игры

Доска состоит из 12 малых лунок (по 6 на каждой стороне) и 2 Калахов — хранилищ игроков. В начале партии в каждую лунку помещается N камней (настраивается). Игроки поочерёдно выбирают лунку на своей стороне, берут все камни и раскладывают их по одному против часовой стрелки:

- **Бонусный ход**: если последний камень попал в собственный Калах — игрок ходит снова
- **Захват**: если последний камень попал в пустую лунку своей стороны — игрок забирает его и все камни из противоположной лунки противника
- **Конец игры**: когда у одного из игроков заканчиваются все камни на его стороне; каждый добирает оставшиеся свои камни в Калах

## 2. Архитектура приложения

Приложение построено по принципу **MVC (Model–View–Controller)**:

```
┌─────────────────────────────────────────┐
│              index.html (View)          │
│         Разметка игрового поля          │
└─────────────────┬───────────────────────┘
                  │ события DOM
┌─────────────────▼───────────────────────┐
│              ui.js (Controller)         │
│   Обработка кликов, анимации, журнал    │
└──────────────────┬──────────────────────┘
                   │ вызовы методов
          ┌────────▼────────┐
          │    game.js      │
          │    (Model)      │
          │  Игровая логика │
          └─────────────────┘
```

### 2.1 Модуль game.js — Игровая логика (Хусейнов А.)

**Класс `BantumiGame`** хранит полное состояние игры:

| Поле | Тип | Описание |
|------|-----|---------|
| `board` | `number[14]` | Массив: лунки 0–5 (П1), Калах 6, лунки 7–12 (П2), Калах 13 |
| `currentPlayer` | `1 \| 2` | Номер игрока, чья очередь |
| `gameOver` | `boolean` | Признак завершения партии |
| `winner` | `0 \| 1 \| 2 \| null` | 0 = ничья |
| `history` | `State[]` | Стек состояний для функции Undo |

**Ключевые алгоритмы:**

*Раскладывание камней:*
```
stones = board[pitIndex]; board[pitIndex] = 0
current = pitIndex
while stones > 0:
    current = (current + 1) mod 14
    if current == opponentKalah: continue   // пропуск Калаха противника
    board[current]++; stones--
```

*Противоположная лунка:* `opposite(i) = 12 - i`
(работает симметрично для обеих сторон: 0↔12, 1↔11, ..., 5↔7)

*Проверка бонусного хода:*
```
if lastStone == myKalah → bonusTurn = true
```

*Проверка захвата:*
```
if ownsPit(current) and board[current] == 1 and board[opposite(current)] > 0:
    board[myKalah] += board[opposite(current)] + 1
    board[opposite(current)] = 0; board[current] = 0
```

### 2.2 Модуль ui.js — Пользовательский интерфейс (Старостин М.)

**Класс `BantumiUI`** управляет DOM:

| Метод | Описание |
|-------|---------|
| `_render()` | Полная перерисовка доски после каждого хода |
| `_createPitEl(idx)` | Создаёт DOM-элемент лунки с обработчиком клика |
| `_onPitClick(idx)` | Вызывает `game.makeMove()`, обновляет интерфейс |
| `_undo()` | Восстанавливает предыдущее состояние из `game.history` |
| `_logMove()` | Добавляет запись в журнал ходов |
| `_showOverlay()` | Показывает экран результата |

### 2.3 Разметка и стили (Старостин М.)

Игровая доска реализована через **CSS Grid** с явным позиционированием:

```css
#kalah-p2 { grid-column: 1; grid-row: 1 / 3; }  /* левый Калах */
#row-p2   { grid-column: 2; grid-row: 1; }       /* лунки П2 */
#row-p1   { grid-column: 2; grid-row: 2; }       /* лунки П1 */
#kalah-p1 { grid-column: 3; grid-row: 1 / 3; }  /* правый Калах */
```
